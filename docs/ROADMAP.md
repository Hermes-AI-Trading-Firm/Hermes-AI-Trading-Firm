## Pipeline Architecture (v1.0)

```
nt8_export/*.csv
    │
    ▼
nt8_connector/          ← file import, duplicate protection
    │
    ▼
research/analytics/     ← equity curve, drawdown, monthly returns, performance
    │
    ▼
research/risk/          ← prop firm compliance, health score, DD checks
    │
    ▼
research/scoring/       ← composite score, grade, hard gates, batch runner
    │
    ▼
research/reporting/     ← Markdown + JSON exports to reports/
    │
    ▼
api/ + dashboard/       ← read-only REST API, local HTML dashboard
    │
    ▼
Decision Queue          ← REVIEW_REQUIRED status, human action prompt
    │
    ▼
⚠️  HUMAN APPROVAL GATE — mandatory before any advancement
```

## Future Phases (Research Pipeline Only)

These extend the read-only research pipeline. No live trading. No broker connections.

### Phase 12 — AI Learning Brain
- `research/ai_learning_brain/` module
- Pattern database: what works, what fails, by regime and asset class
- Fed automatically from `scoring_results` + `overfit_warnings_json`
- Weekly summary report generated by `research/reporting/`

### Phase 13 — Walk-Forward Integration
- Connect `research/walk_forward_engine/` to scoring pipeline
- Populate `walk_forward_pass` / `walk_forward_score` from real WF results
- Re-score after each WF run

### Phase 14 — Monte Carlo Integration
- Connect `research/monte_carlo_engine/` to scoring pipeline
- Populate `monte_carlo_pass` / `monte_carlo_score` from real MC results
- Survival rate surfaced in decision queue

### Phase 15 — Regime Engine Integration
- Connect `research/regime_engine/` to scoring pipeline
- Populate `regime_score` from real Markov/HMM regime analysis
- Regime breakdown surfaced in strategy reports

### Phase 16 — Backtesting Lab Integration
- Structured CSV/JSON backtest import from NinjaTrader Strategy Analyzer
- Populate `backtests` table with real metrics
- Research Rankings and Prop-Firm Candidates panels fully resolve (PF, Sharpe, DD)

### Phase 17 — Forward Testing Journal (Manual)
- Human-driven forward testing journal under `research/forward_testing/`
- Paper trade log, entry/exit notes, hypothesis tracking
- No automation — human records results manually

### Phase 18 — Optimization Lab Integration
- Connect `research/optimization_lab/` to scoring pipeline
- Track optimization runs, parameter sensitivity, IS vs OOS degradation
- Overfit warnings auto-populated from OOS/IS ratio

## Permanent Constraints

The following are **permanently out of scope** for this project:

| Prohibited | Reason |
|------------|--------|
| Live trading | Human approval gate — non-negotiable |
| Broker API connections | No execution layer |
| Order routing | No OMS |
| ATM management | No NinjaTrader ATM control |
| Auto execution | No automated trade triggering |
| Forward testing automation | Manual, human-reviewed only |
